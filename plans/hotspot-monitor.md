# Codex-First Hotspot Monitor Plan

## Summary

The project has two delivery paths:

- Daily reports
  - morning / noon / evening summaries generated by local Codex CLI
- Burst alerts
  - deterministic polling, scoring, and Discord push for timely signals

The runtime must stay agent-light:

- Codex is used for report writing and operator-facing explanations
- Burst decisions are script-driven, not model-driven

## Architecture

Implementation lives under `monitor/` and keeps runtime concerns separated:

- `lib/config.py`
  - config loading and chain mappings
- `lib/state.py`
  - SQLite state for dedupe, cooldown, and recent history
- `lib/scorer.py`
  - hard filters and multi-signal scoring
- `lib/notifier.py`
  - Discord formatting and webhook delivery
- `scripts/collect_snapshot.py`
  - source collection and normalization
- `scripts/burst_scan.py`
  - end-to-end burst scan pipeline
- `scripts/send_report.py`
  - end-to-end report pipeline
- `scripts/run_burst_scan.ps1`
  - Windows wrapper for burst execution
- `scripts/run_send_report.ps1`
  - Windows wrapper for report execution
- `scripts/register_windows_tasks.ps1`
  - Windows Task Scheduler registration plus startup launcher install

## Data Sources

Current implementation uses 7 sources:

- Binance Social Hype
- Binance Trending
- Binance Top Search
- Binance Topic Rush
- Binance Smart Money Inflow
- OKX aggregated buy signals
- OKX hot token snapshots

## Trigger Strategy

### Hard Filters

Assets must satisfy:

- chain in `solana / base / bsc / ethereum`
- valid contract address
- `liquidity_usd >= 100000`
- `holders >= 300`
- `riskControlLevel <= 3`

Drop immediately if blacklist tags exist:

- `honeypot`
- `rug`
- `blacklist`
- `trading_disabled`

Allow report-only, but no burst, if risk-only tags exist:

- `high_bundle`
- `suspicious_dev`
- `extreme_concentration`

### Scoring

- Social Hype new top-5 or rank jump >= 5: `+1`
- Trending / Top Search new entry: `+1`
- Topic Rush inflow >= 50k and >= 1.5x growth: `+2`
- Smart Money Inflow top-10: `+2`
- OKX aggregated signal with >= 3 wallets in 15m: `+3`
- consecutive scan window hit: `+2`
- watchlist hit: `+2`

### Push Tiers

- `0-3`
  - store only
- `4-5`
  - report candidate only
- `6-7`
  - `hot` burst
- `8+`
  - `critical` burst

Burst requires at least one onchain evidence source:

- `okx_signal`
- `smart_money_inflow`

### Cooldown

- default asset cooldown: 90 minutes
- critical upgrade cooldown: 30 minutes
- in-cooldown upgrade allowed only when:
  - score improves by at least 2
  - and a new onchain source appears

### Quiet Hours

- quiet hours: `00:00-07:00 Asia/Shanghai`
- `hot` bursts are suppressed
- `critical` bursts still go out

## Scheduling

Windows deployment is currently the reference setup:

- Task Scheduler
  - `OnchainMonitor_Burst` every 3 minutes
  - `OnchainMonitor_MorningReport` at 08:00
  - `OnchainMonitor_NoonReport` at 12:30
  - `OnchainMonitor_EveningReport` at 20:00
- Startup launcher
  - one immediate burst scan after user logon
